# Свадебный сайт-приглашение

В архиве уже есть рабочий статический сайт:

- `index.html` — структура и тексты сайта.
- `style.css` — весь дизайн: чёрно-белый минимализм, жемчуг, адаптив.
- `script.js` — обратный отсчёт, анимация, форма подтверждения, музыка.
- `assets/dress` — изображения для блока дресс-кода.
- `assets/music` — сюда нужно положить песню.
- `.nojekyll` — служебный файл для GitHub Pages.

## Что заменить под себя

1. **Имена пары**  
   Откройте `index.html` и замените строку `Имя & Имя` на ваши имена.

2. **Точный адрес ЗАГСа**  
   В `index.html` найдите текст `ЗАГС Рудничного района` и допишите точный адрес, когда вспомните.

3. **Фото дресс-кода**  
   Сейчас стоят аккуратные иллюстрации. Чтобы поставить свои фото:
   - положите фото в папку `assets/dress`;
   - в `index.html` замените пути `assets/dress/dress-1.svg`, `dress-2.svg` и так далее на свои файлы, например `assets/dress/photo-1.jpg`.

4. **Песня**  
   Положите аудиофайл в папку `assets/music` и назовите его строго `song.mp3`.  
   После этого кнопка «Включить музыку» заработает. Автозапуск музыки браузеры обычно блокируют, поэтому запуск сделан по кнопке.

5. **Куда приходят ответы гостей**  
   Откройте `script.js` и заполните блок в самом верху:

```js
const RSVP_CONTACTS = {
  PHONE_FOR_SMS: "+79991234567",
  TELEGRAM_USERNAME: "ваш_ник_без_собачки",
  MAX_LINK: "ссылка_на_MAX"
};
```

Если контакты не заполнить, форма всё равно сформирует сообщение, его можно будет скопировать вручную.

## Как выложить на GitHub Pages

1. Зайдите на GitHub и создайте новый репозиторий, например `wedding-invite`.
2. Распакуйте ZIP-архив.
3. Загрузите в репозиторий все файлы и папки из архива:  
   `index.html`, `style.css`, `script.js`, `assets`, `.nojekyll`, `README.md`.
4. Откройте `Settings` → `Pages`.
5. В блоке `Build and deployment` выберите:
   - `Source`: `Deploy from a branch`;
   - `Branch`: `main`;
   - папка: `/root`.
6. Нажмите `Save`.
7. Через 1–3 минуты сайт будет доступен по адресу вида:  
   `https://ваш-логин.github.io/wedding-invite/`

## Важно

Файл `index.html` должен лежать в корне репозитория, не внутри дополнительной папки. Иначе GitHub Pages не откроет сайт правильно.
